### Design note (pure-function purity)
parseCitations(bodyText, evidenceTitles) takes a CALLER-resolved title list
[{ id, title }] rather than the raw evidence array, so the parser stays pure
(no state, no DOM). Title resolution (book.title / entry label / external.title via
the 10.3 refId) is the caller's job in Slice 2. Matching is bidirectional
case-insensitive substring (span contains a title, or a title contains the span).
Segments: { text } | { text, italic } | { text, italic, ids:[..] } (ids>1 = ambiguous).

### Console harness output
```
(match, full title)   PLAIN:"I love " | CITE[e1]:"Their Eyes Were Watching God" | PLAIN:" deeply."
(match, fragment)     PLAIN:"the " | CITE[e1]:"Their Eyes" | PLAIN:" fragment"
(case-insensitive)    PLAIN:"see " | CITE[e2]:"RANGE" | PLAIN:" here"
(no-match -> ital)    PLAIN:"just " | ITAL:"emphasis" | PLAIN:" nothing"
(multi-match)         CITE[x,y]:"e"
(asterisk at edges)   CITE[e2]:"Range" | PLAIN:" vs " | CITE[e3]:"Hidden Potential"
(unclosed -> literal) PLAIN:"a complete " | CITE[e2]:"Range" | PLAIN:" then *unclosed tail"
(empty span ** )      PLAIN:"a**b " | CITE[e2]:"Range"
(no asterisks)        PLAIN:"plain prose only"
(non-string input)    -> [] (empty)
```

accepted as a known residual: a writer who pins an ambiguous citation, leaves, and
returns will see it ambiguous again until re-pinned. Promote to persisted pins if/when
a schema touch is in scope (candidate field: subTheory.citationPins).
